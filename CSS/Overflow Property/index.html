CSS Overflow Properties

The Properties are:

  1. Overflow
    • Visible (Default)

      • overflow: visible;

      This is an example of **visible** overflow. Content spills out of the container's boundaries. This is the **default** browser behavior.

    • Hidden

      • overflow: hidden;

      Any content that goes beyond the container boundary is completely clipped and cannot be scrolled or accessed.

    • Clip (Non-Scrollable Hidden)

      • overflow: clip;

      Content is clipped exactly at the boundary. Crucially, the clipped content is **forbidden from being scrolled** (even programmatically).

    • Clip-Margin (Advanced Clipping)

      • overflow: clip;
      overflow-clip-margin: 15px;

      This uses `overflow: clip;` but the clip boundary is extended **15px outward** from the element's box, allowing content to slightly overhang before being clipped.

    • Scroll

      • overflow: scroll;

      Both vertical and horizontal scrollbars are **always** added to the box, even if they are not strictly needed for the content.

    • Auto (Best Practice)

      • overflow: auto;

      Scrollbars are **only** added if the content actually overflows the container in that specific direction. This is the most common and recommended value.

  2. Overflow-x / Overflow-y (Directional Control)
    • Overflow-x (Horizontal Scroll)

      • overflow-x: scroll; overflow-y: hidden;

      This line of text is intentionally very long (using white-space: nowrap) to force horizontal overflow, resulting in a horizontal scrollbar.
    • Overflow-y (Vertical Scroll)

      • overflow-x: hidden; overflow-y: scroll;

      This content is intentionally tall to cause vertical overflow. The `overflow-y: scroll` property adds a vertical scrollbar.

  3. Overflow-wrap (Handling Long Words)

      4. This property solves the issue of a single, continuous word being wider than the container width, causing layout breakage (horizontal overflow).

    • Problem: Visible Overflow (No Scrollbar)

      • overflow: visible;
      overflow-wrap: normal;

      The long word asdfasdfasdfasdfasdfasdfasdfasdfasdfasdfasdfasdfasdfasdfasdfasdfa ignores the narrow container boundary and spills out.

    • Default Problem (Causes Scrollbar)

      • overflow: auto;
      overflow-wrap: normal;

      If the container has `overflow: auto;`, the long word UnbreakableWordddddddddddddddddddddddddddddddddddddddddddddddddddddd will cause a horizontal scrollbar.

    • Solution 1: Overflow-wrap: break-word

      • overflow-wrap: break-word;

      With break-word, the long word UnbreakableWordddddddddddddddddddddddddddddddddddddddddddddddddddddd is forced to break onto the next line, preventing the horizontal overflow.

    • Solution 2: Overflow-wrap: anywhere

      • overflow-wrap: anywhere;

      anywhere is similar to `break-word` but also contributes to the element's minimum width sizing, making the layout slightly more flexible.
